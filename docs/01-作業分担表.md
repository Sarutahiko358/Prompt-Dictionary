# Web UIコンポーネント辞典 — データ作成ルール

## このプロジェクトについて
HTML/CSS/JavaScriptの要素・プロパティ・APIを網羅したリファレンスアプリ。
データは `data/` フォルダ内のJSONファイルで管理し、`data/_index.json` で有効/無効を切り替える。

## JSONファイル作成時のルール

### フォーマット
data/ フォルダ内の既存JSONファイル（例: data/01-text.json）を必ず参照し、同じ構造で作成すること。

基本構造:
- cat: カテゴリ名
- catIcon: 絵文字1つ
- catDesc: カテゴリ説明（1-2文）
- items: 配列。各項目は以下のキーを持つ
  - name: 要素名（HTMLタグは &lt;div&gt; のようにエスケープ）
  - label: "HTML" / "CSS" / "JS" のいずれか
  - labelClass: "label-html" / "label-css" / "label-js" のいずれか
  - desc: 説明文（2-3文）
  - demo: インラインstyleで自己完結するHTMLデモ。外部CSS/JS依存なし。font-size 0.85rem基準
  - code: コピペで使えるコードスニペット。コメント付き推奨
  - props: 属性/プロパティの配列（4-8個）。各要素は { "name": "コード表記", "desc": "日本語説明" }
  - tags: 分類タグの配列（2-4個）
  - browser: ブラウザ対応状況（「全ブラウザ対応」or 具体バージョン）
  - mdn: MDNのURL（日本語版優先）

### 品質基準
- 1カテゴリ 8-18項目
- demoは必ずインラインstyleで完結させる（外部リソース読込禁止）
- codeはベストプラクティスに従う
- 既存カテゴリとの内容重複は避ける

### 作成後の手順
1. data/ フォルダにJSONファイルを配置
2. data/_index.json の categories 配列に追加し enabled: true にする

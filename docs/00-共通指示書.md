# Web UIコンポーネント辞典 — データ作成ルール v2

## 目的
HTML/CSS/JavaScript の要素・プロパティ・APIを網羅した
リファレンスアプリのデータJSONを作成する。

## 出力形式
1つのJSONファイルとして出力。フォーマットは以下に厳密に従う:

```json
{
  "cat": "カテゴリ名",
  "catIcon": "絵文字1つ",
  "catDesc": "カテゴリの説明文（1-2文）",
  "items": [
    {
      "name": "要素名（HTMLタグはエスケープ: &lt;div&gt;）",
      "label": "HTML" | "CSS" | "JS",
      "labelClass": "label-html" | "label-css" | "label-js",
      "desc": "説明文（2-3文。用途・特徴・注意点を簡潔に）",
      "demo": "HTMLデモコード（インラインstyleで自己完結。外部依存なし）",
      "code": "コードスニペット（コピー用。ベストプラクティス準拠）",
      "props": [
        { "name": "属性/プロパティ名", "desc": "簡潔な説明" }
      ],
      "tags": ["タグ1", "タグ2"],
      "browser": "ブラウザ対応状況（簡潔に）",
      "mdn": "MDNのURL"
    }
  ]
}
